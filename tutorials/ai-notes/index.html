AI 学习笔记

面向零基础新手，系统性介绍AI领域的各项概念与原理
涵盖16大主题章节，包括智能体、知识库、工具技能等前沿内容
深入浅出，帮助你快速理解AI的核心知识

第一章：AI基础概念

什么是人工智能（AI）？

人工智能（Artificial Intelligence，AI）是指让计算机模拟人类智能行为的技术。简单来说，就是让机器能够像人一样"思考"、"学习"和"解决问题"。

感知

识别图像、语音、文字

推理

逻辑分析、决策判断

学习

从数据中获取知识

交互

理解语言、对话交流

AI的三个层次

  1. 弱人工智能（Weak AI）：专注于特定任务的AI，如人脸识别、语音助手。这是目前我们实际应用的AI。
  2. 强人工智能（Strong AI）：具备与人类同等智能水平的AI，能处理各种复杂任务。目前仍在研究中。
  3. 超人工智能（Super AI）：超越人类智能的AI。这是未来的理论概念。

常见误区

AI ≠ 机器人。AI是软件层面的智能算法，机器人是硬件载体。AI可以存在于任何设备中（手机、电脑、云端服务器），不一定需要物理形态。

第二章：机器学习与深度学习

什么是机器学习（ML）？

机器学习（Machine Learning）是AI的核心分支，让计算机从数据中自动学习规律，而不是人工编写所有规则。

简单理解

传统编程：人类编写规则 → 计算机执行
机器学习：计算机从数据中学习规则 → 自动改进

机器学习的三种类型

类型 特点 应用场景
监督学习 有标签数据，告诉机器正确答案 图像分类、垃圾邮件识别、预测房价
无监督学习 无标签数据，让机器自己发现规律 用户分群、数据降维、异常检测
强化学习 通过试错获得奖励，学习最优策略 游戏AI、自动驾驶、机器人控制

监督学习详解

监督学习是最常见的机器学习方式。就像老师教学生：老师给题目和答案，学生通过对比学习。

训练数据（输入+标签）
→
学习算法
→
模型
→
预测新数据

监督学习的两大任务：

  • 分类（Classification）：预测离散类别，如"是/否"、"猫/狗"
  • 回归（Regression）：预测连续数值，如房价、温度

什么是深度学习（DL）？

深度学习（Deep Learning）是机器学习的子领域，使用多层神经网络来学习数据的复杂特征。

层级关系

人工智能 ⊃ 机器学习 ⊃ 深度学习
（AI包含ML，ML包含DL）

为什么叫"深度"学习？

因为神经网络有很多"层"。传统神经网络可能只有2-3层，深度学习网络可能有几十甚至上百层。

输入层

接收原始数据

隐藏层（多层）

逐层提取特征

输出层

给出结果

深度学习的优势

  • 能自动学习复杂特征，无需人工特征工程
  • 处理图像、语音、自然语言效果极佳
  • 大规模数据和算力支持后性能持续提升

常见深度学习架构

架构 特点 应用
CNN（卷积神经网络） 擅长处理图像，能识别空间模式 图像分类、目标检测、人脸识别
RNN（循环神经网络） 擅长处理序列数据，有记忆能力 语音识别、文本翻译、时间序列预测
Transformer 并行处理序列，注意力机制 GPT、BERT、大语言模型

第三章：模型

什么是模型？

模型（Model）是机器学习的核心产出，是一个数学函数或算法，能将输入数据转换为预测输出。

通俗理解

模型就像一个"黑盒子"：
输入 → [模型] → 输出
比如输入一张图片，模型输出"这是猫"

模型的本质

模型内部是大量的参数（Parameters）——就是数字。这些参数决定了模型如何处理输入。

  • 一个简单模型可能有几百个参数
  • GPT-3有1750亿个参数
  • GPT-4参数量更大（具体数字未公开）

常见模型类型

类型 说明 例子
判别式模型 直接学习输入→输出的映射 分类器、回归模型
生成式模型 学习数据分布，能生成新数据 GPT、Stable Diffusion
大语言模型（LLM） 大规模文本生成模型 ChatGPT、Claude、Gemini

大语言模型（LLM）详解

大语言模型（Large Language Model）是基于Transformer架构的生成式模型，能理解和生成人类语言。

LLM的工作原理

1. 接收文本输入
2. 将文本切分为token（词元）
3. 通过多层Transformer处理
4. 预测下一个token的概率
5. 输出最可能的文本序列

Token是什么？

Token是文本的最小处理单位。可以是：

  • 一个词（如"apple"）
  • 一个字（如"你"）
  • 一个词的一部分（如"ing"）

例如："我爱学习AI" 可能被切分为：["我", "爱", "学习", "AI"] 共4个tokens。

第四章：训练

什么是训练？

训练（Training）是让模型从数据中学习的过程。通过反复调整模型参数，使其能准确完成任务。

类比理解

训练就像学生做练习题：
做题 → 对照答案 → 发现错误 → 调整方法 → 再做题 → ...
循环直到正确率足够高

训练的核心流程

准备数据
→
初始化模型
→
前向传播
→
计算损失
→
反向传播
→
更新参数

关键概念详解

1. 损失函数（Loss Function）

损失函数衡量模型预测与真实答案的差距。损失越小，模型越好。

  • 分类任务常用：交叉熵损失（Cross-Entropy）
  • 回归任务常用：均方误差（MSE）

2. 优化器（Optimizer）

优化器决定如何更新模型参数来降低损失。

  • SGD：随机梯度下降，最基础
  • Adam：自适应学习率，效果稳定，最常用

3. 学习率（Learning Rate）

学习率控制每次参数更新的幅度。

  • 太大：不稳定，可能跳过最优解
  • 太小：训练太慢
  • 需要调优找到合适值

批量大小（Batch Size）

一次训练使用多少样本：
• 小批量：更频繁更新，内存占用小
• 大批量：更新更稳定，但内存占用大

训练的常见问题

问题 现象 解决方法
过拟合 训练数据表现好，新数据表现差 更多数据、正则化、Dropout
欠拟合 训练数据和新数据都表现差 增加模型复杂度、更多训练
梯度消失 深层网络无法有效学习 ReLU激活函数、Batch Normalization

第五章：推理

什么是推理？

推理（Inference）是使用已训练好的模型处理新数据，得到预测结果的过程。

简单理解

训练 = 制作工具
推理 = 使用工具

推理与训练的区别

阶段 目的 计算量 更新参数
训练 学习知识，调整参数 巨大（需要GPU集群） 会
推理 应用知识，输出结果 较小（单设备即可） 不会

推理的关键指标

  • 延迟（Latency）：处理单个请求的时间
  • 吞吐量（Throughput）：单位时间处理的请求数
  • 内存占用：运行模型需要的内存

推理部署方式

  1. 本地部署：模型在用户设备运行（如手机端AI）
  2. 云端部署：模型在服务器运行，用户通过API调用
  3. 边缘部署：模型在边缘设备运行（如智能摄像头）

第六章：RAG（检索增强生成）

什么是RAG？

RAG（Retrieval-Augmented Generation）是一种结合信息检索和生成模型的技术，让AI能利用外部知识库回答问题。

为什么需要RAG？

LLM的局限：
• 训练后知识固定，无法获取新信息
• 可能产生错误信息（幻觉）
• 无法访问私有数据

RAG解决：检索相关文档 → 增强上下文 → 生成准确回答

RAG的工作流程

用户提问
→
检索相关文档
→
构建增强上下文
→
LLM生成回答

RAG的核心组件

  1. 文档库：存储知识的外部数据源
  2. 向量数据库：存储文档的向量表示，支持相似检索
  3. Embedding模型：将文本转换为向量
  4. LLM：基于检索内容生成回答

RAG的优势

  • 回答更准确，减少幻觉
  • 能利用最新数据
  • 可引用数据来源
  • 适合企业私有知识库

常见向量数据库

Pinecone、Weaviate、Milvus、Chroma、Qdrant等
用于存储和检索文档向量

第七章：向量与标量

什么是标量？

标量（Scalar）是一个单独的数值，只有大小，没有方向。

  • 温度：25°C
  • 重量：5kg
  • 价格：100元

在AI中，单个参数、单个神经元输出都是标量。

什么是向量？

向量（Vector）是一组有序的数值，既有大小也有方向。

例子

二维向量：[3, 4] 表示坐标系中的点
三维向量：[1, 2, 3] 表示空间中的点
高维向量：[0.1, 0.2, ..., 0.768] 表示768维空间

向量在AI中的作用

Embedding（嵌入）是将文本、图像等转换为向量表示的技术。

为什么用向量？

向量能表示语义相似性：
• "猫"和"狗"的向量距离近（都是动物）
• "猫"和"汽车"的向量距离远
• 计算机通过向量理解语义关系

向量运算

运算 用途 AI应用
向量加法 合并信息 组合多个词的语义
向量点积 计算相似度 文档检索、推荐系统
向量余弦相似度 衡量方向相似 语义相似度计算

向量维度

  • 词向量：通常100-300维
  • BERT Embedding：768维
  • GPT-3 Embedding：12288维

维度越高，能表示的信息越丰富，但计算成本也越高。

第八章：神经网络核心概念

神经网络基础

神经网络是深度学习的基础架构，模拟人脑神经元连接的方式处理信息。

神经元（Neuron）

单个神经元的工作流程：

  1. 接收多个输入
  2. 加权求和（权重是可学习的参数）
  3. 加上偏置（bias）
  4. 通过激活函数
  5. 输出结果
输入 x₁, x₂, x₃
→
加权 w₁x₁ + w₂x₂ + w₃x₃
→
加偏置 +b
→
激活函数
→
输出 y

激活函数（Activation Function）

激活函数给神经网络引入非线性，使其能学习复杂模式。

函数 特点 用途
ReLU 简单高效：x>0输出x，否则输出0 最常用，深层网络
Sigmoid 输出0-1，适合概率 输出层分类
Tanh 输出-1到1 隐藏层
Softmax 多分类输出概率分布 多分类问题

反向传播（Backpropagation）

反向传播是训练神经网络的核心算法：

  • 前向传播：数据从输入层到输出层，得到预测
  • 计算损失：对比预测与真实值
  • 反向传播：从输出层向输入层计算梯度
  • 更新参数：根据梯度调整权重和偏置

第九章：常见模型架构

CNN（卷积神经网络）

CNN专门处理图像数据，能自动识别图像中的特征。

CNN核心组件

  • 卷积层：用滤波器提取图像特征
  • 池化层：降低特征维度，减少计算量
  • 全连接层：将特征映射到输出类别
输入图像
→
卷积层提取特征
→
池化层降维
→
全连接层分类
→
输出结果

Transformer

Transformer是目前最主流的架构，GPT、BERT等大模型都基于它。

Transformer的核心：注意力机制

注意力机制（Attention）让模型能关注输入中最相关的部分。

简单理解

阅读句子"我喜欢编程，特别是Python"
处理"Python"时，注意力会关注"编程"
因为它们语义相关

Self-Attention（自注意力）

让序列中的每个元素都能关注其他元素：

  • 计算Query（查询）、Key（键）、Value（值）
  • Query与Key计算相似度（注意力分数）
  • 用注意力分数加权Value
  • 得到融合上下文信息的表示

Transformer架构

  • Encoder：理解输入，BERT使用
  • Decoder：生成输出，GPT使用
  • Encoder-Decoder：翻译任务，T5使用

第十章：实用概念

Prompt Engineering

提示词工程是设计有效输入，引导LLM产生理想输出的技术。

常见技巧

  • 明确指令：清晰说明任务要求
  • 提供示例：给出想要的输出格式
  • 角色设定："你是一个专家..."
  • 分步骤：让模型逐步思考
  • 思维链："让我们一步步思考..."

Prompt示例

"你是一位Python专家。请解释这段代码的功能，使用简单语言，适合初学者理解。
代码：[代码内容]
请分三个部分回答：1. 功能概述 2. 详细解释 3. 使用场景"

Fine-tuning（微调）

微调是在预训练模型基础上，用特定领域数据进一步训练，使其适应特定任务。

过程

预训练模型（通用知识）+ 特定数据训练 → 专用模型

微调 vs 预训练

阶段 数据量 计算资源 目的
预训练 海量（TB级） 巨大（数千GPU） 学习通用知识
微调 较少（GB级） 较小（几十GPU） 适应特定任务

Temperature（温度）

温度参数控制LLM输出的随机性。

  • 低温度（0.1-0.3）：输出更确定、一致，适合事实性问题
  • 高温度（0.7-1.0）：输出更有创意、多样，适合创意写作

选择建议

• 编程问题：temperature=0.2
• 创意写作：temperature=0.8
• 一般对话：temperature=0.5

第十一章：AI智能体（Agent）

什么是AI智能体？

AI智能体（Agent）是一个能够感知环境、做出决策、执行行动以达成目标的AI系统。它不仅仅是回答问题，而是能主动完成任务。

简单理解

普通AI：你问它答，被动响应
AI智能体：你给目标，它自主规划并执行，主动完成

Agent的核心能力

感知

理解用户指令和环境信息

规划

拆解任务，制定执行策略

行动

调用工具，执行具体操作

反思

评估结果，调整策略

Agent的工作流程

接收目标
→
任务拆解
→
选择工具
→
执行操作
→
观察结果
→
调整继续

Agent的关键组件

组件 作用 例子
大脑（Brain） 核心LLM，负责理解和决策 GPT-4、Claude、Gemini
记忆（Memory） 存储历史对话和知识 短期记忆、长期记忆、向量数据库
工具（Tools） 执行具体操作的能力 搜索、代码执行、文件操作
规划（Planning） 任务分解和执行策略 ReAct、Plan-and-Execute

常见Agent类型

  • 对话型Agent：ChatGPT、Claude等，主要进行对话交流
  • 任务型Agent：能执行具体任务，如写代码、搜索资料、生成报告
  • 多Agent系统：多个Agent协作完成复杂任务，如CrewAI
  • 自主Agent：能长期自主运行，如AutoGPT

常见Agent框架

• LangChain：构建Agent应用的主流框架
• CrewAI：多Agent协作框架
• AutoGPT：自主Agent先驱
• AgentGPT：网页版自主Agent
• OpenAI Assistants API：官方Agent解决方案

第十二章：工具（Tools）与技能（Skills）

什么是工具？

工具（Tools）是Agent可以调用的外部功能，让AI能执行实际操作，而不仅仅是生成文本。

为什么需要工具？

LLM本身只能生成文本，无法：
• 搜索最新信息
• 执行代码
• 访问数据库
• 操作文件

工具让AI突破这些限制

常见工具类型

工具类型 功能 应用场景
搜索工具 搜索网络信息 获取实时数据、查找资料
代码执行 运行Python等代码 数据分析、计算、自动化
文件操作 读写文件 文档处理、数据导入导出
API调用 调用外部服务 获取天气、发送邮件、数据库查询
知识检索 从知识库检索 RAG、企业知识查询

Function Calling（函数调用）

函数调用是LLM调用外部工具的技术机制。

用户请求
→
LLM判断需要工具
→
生成函数调用参数
→
执行函数
→
返回结果给LLM
→
生成最终回答

函数调用示例

用户："今天北京天气怎么样？"
LLM：调用get_weather(city="北京")
函数返回：{"temperature": 25, "weather": "晴"}
LLM："北京今天天气晴朗，温度25°C"

什么是技能（Skills）？

技能（Skills）是AI Agent的特定能力模块，类似于人类的"专业技能"。

类比理解

人：掌握多个技能（编程、写作、设计）
Agent：配置多个技能模块（代码生成、数据分析、文档撰写）

技能的特点

  • 模块化：可以独立配置和启用/禁用
  • 专业化：针对特定领域优化
  • 可扩展：可以添加新技能
  • 可组合：多个技能协作完成任务

常见技能示例

技能领域 具体技能 应用
编程技能 代码生成、调试、重构 软件开发、自动化脚本
分析技能 数据分析、图表生成、报告撰写 商业分析、科学研究
写作技能 文章创作、编辑润色、翻译 内容创作、文档处理
搜索技能 网络搜索、信息提取、知识检索 调研、信息收集
设计技能 图像生成、UI设计建议 创意设计、产品开发

技能 vs 工具

• 工具：底层能力，如"搜索"、"执行代码"
• 技能：上层组合，整合工具+知识+策略

例如："数据分析技能"包含：
- 代码执行工具 + 统计知识 + 分析流程策略

第十三章：知识库与向量数据库

什么是知识库？

知识库（Knowledge Base）是存储结构化或非结构化知识的系统，让AI能访问专业知识、企业数据、实时信息。

为什么需要知识库？

LLM的问题：
• 知识有限（训练数据截止）
• 无法访问企业私有数据
• 可能产生错误信息（幻觉）

知识库解决：
• 提供最新、准确的专业知识
• 让AI"知道"企业内部信息

知识库的类型

类型 特点 应用
文档库 PDF、Word、网页等文档集合 企业文档查询、政策解读
结构化数据库 SQL数据库，有明确结构 客户信息、订单数据、产品目录
向量数据库 存储向量表示，支持语义搜索 RAG、相似内容检索
知识图谱 实体和关系的网络结构 复杂关系查询、推理

知识库建设流程

收集数据
→
清洗处理
→
分块切分
→
向量编码
→
存储入库
→
检索使用

向量数据库详解

向量数据库是专门存储和检索向量Embedding的数据库，支持语义相似性搜索。

传统搜索 vs 向量搜索

搜索方式 原理 特点
关键词搜索 匹配文字相同 "苹果"只能找到包含"苹果"的文档
向量搜索 匹配语义相似 "苹果"能找到"水果"、"iOS设备"等相关内容

向量搜索的优势

• 理解语义，不是简单匹配文字
• 能找到意思相近的内容
• 支持跨语言搜索
• 处理模糊、抽象的查询

常见向量数据库

  • Pinecone：云端托管，易于使用，适合生产环境
  • Weaviate：开源，支持多种数据类型，功能丰富
  • Milvus：开源，高性能，适合大规模场景
  • Chroma：轻量级开源，适合开发测试
  • Qdrant：开源，高效，支持过滤

向量检索流程

  1. 查询向量化：将用户查询转换为向量
  2. 相似度计算：与库中向量计算距离
  3. 排序筛选：返回最相似的N个结果
  4. 返回原文：返回对应的原始文本

第十四章：上下文（Context）与记忆

什么是上下文？

上下文（Context）是LLM处理请求时可用的背景信息，包括历史对话、检索内容、系统提示等。

上下文窗口

LLM一次能处理的文本长度限制：
• GPT-3.5：4K tokens
• GPT-4：8K-32K tokens
• Claude：100K-200K tokens
• GPT-4 Turbo：128K tokens

上下文的组成

系统提示

角色设定、行为规范

历史对话

之前的交流记录

检索内容

RAG获取的相关文档

用户输入

当前的请求

上下文管理策略

  • 滑动窗口：只保留最近N轮对话
  • 摘要压缩：用LLM总结历史，减少token占用
  • 选择性保留：只保留关键信息
  • 向量检索：需要时检索相关历史

Agent的记忆系统

记忆（Memory）是Agent存储和利用历史信息的能力。

记忆的类型

记忆类型 特点 存储方式
短期记忆 当前对话上下文 在LLM上下文窗口中
长期记忆 跨对话的持久知识 向量数据库、文件存储
工作记忆 当前任务相关信息 临时存储，任务完成可清除
语义记忆 概念和知识 知识库、知识图谱

记忆的作用

  • 记住用户偏好：下次对话能个性化响应
  • 积累知识：学习新信息，逐步成长
  • 保持一致性：多轮对话中不重复、不矛盾
  • 任务延续：长时间任务的进度跟踪

记忆存储技术

• 向量数据库：语义检索记忆
• 键值存储：结构化记忆
• 文件系统：文档型记忆
• 时间序列：按时间组织记忆

第十五章：高级提示技术

Chain of Thought（思维链）

思维链是一种让LLM逐步展示推理过程的提示技术，能显著提高复杂问题的准确率。

基本方法

在Prompt中加入："让我们一步步思考"
或者："请展示你的推理过程"

思维链的效果

问题
→
步骤1：理解问题
→
步骤2：分解任务
→
步骤3：逐步推理
→
步骤4：验证答案
→
最终结论

思维链的应用

  • 数学推理：逐步计算，减少错误
  • 逻辑分析：展示推理过程
  • 复杂决策：权衡多个因素
  • 问题诊断：系统性排查

Zero-shot vs Few-shot

零样本（Zero-shot）：不给示例，让LLM直接完成任务。

少样本（Few-shot）：提供几个示例，让LLM学习模式。

示例对比

Zero-shot：
"将以下句子翻译成法语：你好"

Few-shot：
"将以下句子翻译成法语：
你好 → Bonjour
谢谢 → Merci
再见 → Au revoir
我爱你 → ?"

何时使用Few-shot

  • 任务格式特殊，需要示范
  • 输出风格需要统一
  • LLM对任务不够熟悉
  • 需要特定格式输出

ReAct（推理+行动）

ReAct是让Agent交替进行推理和行动的技术，边思考边执行。

ReAct流程

  1. Thought：思考下一步做什么
  2. Action：执行具体操作
  3. Observation：观察结果
  4. 循环：重复直到完成

ReAct示例

任务："查找北京今天的天气"
Thought: 我需要搜索北京的天气信息
Action: search("北京天气")
Observation: 北京今天晴，25°C
Thought: 我已获得答案
Action: finish("北京今天天气晴朗，温度25°C")

第十六章：AI局限与挑战

幻觉问题（Hallucination）

幻觉是LLM生成虚假、不准确信息的现象，是AI应用的主要风险。

幻觉的类型

类型 表现 风险
事实幻觉 编造不存在的事实 "2025年火星已有人居住"
来源幻觉 引用不存在的来源 "根据《AI发展报告2024》..."
数字幻觉 编造统计数据 "研究表明85%的用户..."
逻辑幻觉 推理过程出错 错误的因果结论

减少幻觉的方法

  • 使用RAG：检索真实信息增强回答
  • 要求引用来源：让AI说明信息出处
  • 人工验证：重要信息需要核实
  • 限定范围：指定只使用已知信息
  • 多模型交叉验证：对比不同模型输出

安全建议

• 不要完全信任AI的医疗、法律建议
• 重要决策前核实关键信息
• 教育用户了解AI局限性
• 建立信息审核机制

对齐问题（Alignment）

对齐是让AI行为符合人类价值观和期望的技术挑战。

对齐的维度

  • 指令对齐：AI正确理解用户意图
  • 价值对齐：AI行为符合道德伦理
  • 安全对齐：AI不产生有害内容
  • 真实对齐：AI提供准确信息

RLHF（基于人类反馈的强化学习）

RLHF是训练AI对齐的主要方法：

  1. 预训练：学习语言能力
  2. 监督微调：学习遵循指令
  3. 奖励模型：人类评价AI输出质量
  4. 强化学习：优化AI生成高奖励内容

为什么需要人类反馈？

单纯预训练的模型：
• 可能输出有害内容
• 不一定遵循指令
• 价值观与人类不符

人类反馈让AI学会"好的回答"

多模态（Multimodal）

多模态AI能同时处理多种类型的数据：文本、图像、音频、视频等。

多模态的优势

  • 理解更丰富：结合视觉和语言信息
  • 应用更广：图像分析、视频理解、语音识别
  • 交互更自然：支持多种输入方式

常见多模态模型

模型 能力 应用
GPT-4 Vision 理解图像内容 图像问答、视觉分析
Claude 3 文本+图像处理 文档分析、图表解读
Gemini 文本+图像+视频+音频 全面多模态应用

学习路线建议

新手入门路径

  1. 基础概念：理解AI、ML、DL的关系和基本原理
  2. 编程基础：学习Python，这是AI领域的主要语言
  3. 数学基础：线性代数、概率统计、微积分基础
  4. 实践入门：使用现成工具（如ChatGPT API）体验AI
  5. 深入学习：学习机器学习算法、神经网络原理
  6. 项目实践：完成实际项目，巩固知识
