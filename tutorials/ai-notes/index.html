第一章：AI基础概念

什么是人工智能（AI）？

人工智能（Artificial Intelligence，AI）是指让计算机模拟人类智能行为的技术。简单来说，就是让机器能够像人一样"思考"、"学习"和"解决问题"。

感知

识别图像、语音、文字

推理

逻辑分析、决策判断

学习

从数据中获取知识

交互

理解语言、对话交流

AI的三个层次

  1. 弱人工智能（Weak AI）：专注于特定任务的AI，如人脸识别、语音助手。这是目前我们实际应用的AI。
  2. 强人工智能（Strong AI）：具备与人类同等智能水平的AI，能处理各种复杂任务。目前仍在研究中。
  3. 超人工智能（Super AI）：超越人类智能的AI。这是未来的理论概念。

常见误区

AI ≠ 机器人。AI是软件层面的智能算法，机器人是硬件载体。AI可以存在于任何设备中（手机、电脑、云端服务器），不一定需要物理形态。

第二章：机器学习与深度学习

什么是机器学习（ML）？

机器学习（Machine Learning）是AI的核心分支，让计算机从数据中自动学习规律，而不是人工编写所有规则。

简单理解

传统编程：人类编写规则 → 计算机执行
机器学习：计算机从数据中学习规则 → 自动改进

机器学习的三种类型

类型 特点 应用场景
监督学习 有标签数据，告诉机器正确答案 图像分类、垃圾邮件识别、预测房价
无监督学习 无标签数据，让机器自己发现规律 用户分群、数据降维、异常检测
强化学习 通过试错获得奖励，学习最优策略 游戏AI、自动驾驶、机器人控制

监督学习详解

监督学习是最常见的机器学习方式。就像老师教学生：老师给题目和答案，学生通过对比学习。

训练数据（输入+标签）
→
学习算法
→
模型
→
预测新数据

监督学习的两大任务：

  • 分类（Classification）：预测离散类别，如"是/否"、"猫/狗"
  • 回归（Regression）：预测连续数值，如房价、温度

监督学习的应用实例

垃圾邮件识别：
• 训练数据：10万封邮件（标记"垃圾"或"正常"）
• 模型学习：垃圾邮件的特征（关键词、发件人）
• 应用：新邮件自动判断是否垃圾

房价预测：
• 训练数据：历史房价（面积、位置、房价）
• 模型学习：面积/位置与房价的关系
• 应用：输入房屋信息，预测房价

无监督学习详解

无监督学习（Unsupervised Learning）是让机器在没有标签的数据中自己发现规律和结构。就像让学生在没有老师指导的情况下自己探索知识。

为什么叫"无监督"？

监督学习：有老师告诉对错（有标签）
无监督学习：没有老师，机器自己探索（无标签）

数据只有输入，没有标准答案。机器的任务是找出数据内部的规律。

无监督学习的主要任务

1. 聚类（Clustering）——自动分组

聚类是把相似的数据自动归为一组，发现数据中的自然分组。

聚类的通俗理解

想象你有一堆照片，不知道分类。聚类算法会自动：
• 把人物照片放在一起
• 把风景照片放在一起
• 把动物照片放在一起

机器不知道什么是"人物"、"风景"，但它能发现照片之间的相似性，自动分组。

常见聚类算法

算法 原理 特点
K-Means 把数据分成K个组，每组围绕一个中心点 最简单常用，适合圆形分布数据
层次聚类 逐步合并或分拆，形成树状结构 能看到数据的层级关系
DBSCAN 基于密度，把密集区域归为一组 能发现任意形状的簇，自动识别噪声

聚类的应用场景

  • 客户分群：电商平台把用户分成"价格敏感"、"品质追求"、"潮流追随"等群体
  • 新闻分类：自动把相似主题的新闻归为一类
  • 社交网络分析：发现朋友圈中的不同社交圈子
  • 基因分析：根据基因表达数据发现疾病类型

2. 降维（Dimensionality Reduction）——简化数据

降维是把高维数据压缩到低维，保留关键信息，去掉冗余。

什么是高维数据？

每个数据点有很多属性（维度）：
• 用户数据：年龄、性别、收入、职业、爱好...（几十个维度）
• 图像：每个像素是一个维度（一张图片可能有上万个维度）

维度太多的问题：
• 计算复杂，处理慢
• 信息冗余，很多维度相关
• 难以可视化理解

降维解决这个问题：保留核心信息，压缩到少数维度

常见降维算法

算法 原理 应用
PCA（主成分分析） 找到数据变化最大的方向，投影到这些方向 数据压缩、可视化、去噪
t-SNE 保持局部相似性，用于可视化 高维数据可视化展示
UMAP 比t-SNE更快，保持全局结构 大规模数据可视化

降维的应用场景

  • 数据可视化：把100维数据降到2维，画在图表上直观展示
  • 图像压缩：减少图像存储空间，保留关键特征
  • 特征提取：从大量特征中提取核心特征，简化模型
  • 去噪处理：去掉数据中的噪声维度

3. 异常检测（Anomaly Detection）——发现异常

异常检测是找出与正常数据明显不同的异常数据。

异常检测的价值

• 欺诈检测：发现异常交易行为（银行卡盗刷）
• 故障预警：发现设备运行的异常状态
• 入侵检测：发现网络异常流量攻击
• 数据清洗：发现数据中的错误异常值

无监督学习的挑战

  • 结果不确定：没有标准答案，聚类结果可能不唯一
  • 评估困难：不知道分群是否正确，需要人工验证
  • 解释性弱：机器发现的规律，人类可能难以理解

强化学习详解

强化学习（Reinforcement Learning）是让AI通过不断尝试和反馈来学习最优策略。就像训练宠物：表现好给奖励，表现差不奖励，逐渐学会正确行为。

类比理解

训练狗狗：
• 狗坐下 → 给零食（奖励） → 狗记住坐下能得零食
• 狗乱跑 → 不给零食（无奖励） → 狗知道乱跑没好处
• 多次尝试后 → 狗学会"听到指令就坐下"

强化学习就是这个原理：AI尝试各种行为，根据奖励反馈调整策略。

强化学习的核心概念

状态（State）

当前环境的情况
（如：游戏中角色的位置）

动作（Action）

AI能做的事情
（如：向上、向下、攻击）

奖励（Reward）

行为好坏的反馈
（如：得分、扣分）

策略（Policy）

在什么状态做什么动作
（AI学到的决策规则）

强化学习的循环过程

观察状态
→
选择动作
→
执行动作
→
获得奖励
→
更新策略
→
进入新状态

探索与利用的平衡

强化学习面临一个核心难题：该尝试新方法（探索），还是用已知的最好方法（利用）？

探索 vs 利用

探索：尝试新的、未知的行为
• 好处：可能发现更好的方法
• 风险：可能暂时表现不好

利用：使用已知的最佳行为
• 好处：稳定获得好结果
• 风险：错过更好的方法

好的强化学习需要平衡两者：既探索新可能，又利用已知最优。

常见强化学习算法

算法 原理 应用
Q-Learning 建立表格记录每个(状态,动作)的预期奖励 简单环境、游戏AI
DQN 用神经网络代替Q表，处理复杂环境 Atari游戏、复杂游戏
Policy Gradient 直接优化动作策略，不是估计奖励值 连续动作空间、机器人
PPO 限制策略更新幅度，训练稳定 ChatGPT的RLHF训练

强化学习的经典应用

  • AlphaGo：通过自我对弈，学会围棋最优策略，战胜人类冠军
  • 游戏AI：马里奥、星际争霸、DOTA等游戏AI通过强化学习通关
  • 自动驾驶：学习在复杂环境中安全驾驶
  • 机器人控制：学习行走、抓取等动作
  • 推荐系统：根据用户反馈优化推荐策略

AlphaGo的突破

AlphaGo的学习过程：
1. 监督学习：学习人类高手棋谱
2. 强化学习：自我对弈，探索更好的策略
3. 发现人类未知的招法，超越人类水平

这是强化学习的经典成功案例。

强化学习的挑战

  • 训练慢：需要大量尝试才能学会
  • 奖励设计难：奖励设置不好，AI可能学错
  • 现实应用难：真实环境复杂，不能无限尝试

什么是深度学习（DL）？

深度学习（Deep Learning）是机器学习的子领域，使用多层神经网络来学习数据的复杂特征。

层级关系

人工智能 ⊃ 机器学习 ⊃ 深度学习
（AI包含ML，ML包含DL）

为什么叫"深度"学习？

因为神经网络有很多"层"。传统神经网络可能只有2-3层，深度学习网络可能有几十甚至上百层。

输入层

接收原始数据

隐藏层（多层）

逐层提取特征

输出层

给出结果

深度学习的优势

  • 能自动学习复杂特征，无需人工特征工程
  • 处理图像、语音、自然语言效果极佳
  • 大规模数据和算力支持后性能持续提升

常见深度学习架构

架构 特点 应用
CNN（卷积神经网络） 擅长处理图像，能识别空间模式 图像分类、目标检测、人脸识别
RNN（循环神经网络） 擅长处理序列数据，有记忆能力 语音识别、文本翻译、时间序列预测
Transformer 并行处理序列，注意力机制 GPT、BERT、大语言模型

第三章：模型

什么是模型？

模型（Model）是机器学习的核心产出，是一个数学函数或算法，能将输入数据转换为预测输出。

通俗理解

模型就像一个"黑盒子"：
输入 → [模型] → 输出
比如输入一张图片，模型输出"这是猫"

模型的本质

模型内部是大量的参数（Parameters）——就是数字。这些参数决定了模型如何处理输入。

  • 一个简单模型可能有几百个参数
  • GPT-3有1750亿个参数
  • GPT-4参数量更大（具体数字未公开）

常见模型类型

类型 说明 例子
判别式模型 直接学习输入→输出的映射 分类器、回归模型
生成式模型 学习数据分布，能生成新数据 GPT、Stable Diffusion
大语言模型（LLM） 大规模文本生成模型 ChatGPT、Claude、Gemini

大语言模型（LLM）详解

大语言模型（Large Language Model）是基于Transformer架构的生成式模型，能理解和生成人类语言。

LLM的工作原理

1. 接收文本输入
2. 将文本切分为token（词元）
3. 通过多层Transformer处理
4. 预测下一个token的概率
5. 输出最可能的文本序列

Token是什么？

Token是文本的最小处理单位。可以是：

  • 一个词（如"apple"）
  • 一个字（如"你"）
  • 一个词的一部分（如"ing"）

例如："我爱学习AI" 可能被切分为：["我", "爱", "学习", "AI"] 共4个tokens。

模型大小与参数量

参数量是衡量模型规模的指标，参数越多，模型容量越大，通常效果更好，但成本也更高。

主流LLM的参数量对比

模型 参数量 显存需求（FP16） 特点
GPT-2 Small 117M（1.17亿） ~0.5GB 早期模型，适合入门学习
GPT-2 Large 1.5B（15亿） ~3GB 可在消费级显卡运行
Llama-2-7B 7B（70亿） ~14GB 开源主流，RTX 4090可运行
Llama-2-13B 13B（130亿） ~26GB 效果更好，需要高端显卡
Mistral-7B 7B ~14GB 效率高，小模型中的佼佼者
Qwen-7B 7B ~14GB 中文能力强，国产开源
Llama-2-70B 70B ~140GB 顶级开源，需要专业GPU
GPT-3 175B ~350GB 早期大模型标杆
GPT-4 未公开（推测万亿级） 需要集群 当前最强闭源模型之一

参数量与效果的关系

参数越多 → 效果越好（但不是线性关系）：
• 7B模型：适合简单任务，日常对话
• 13B模型：效果明显提升，复杂任务
• 70B模型：接近GPT-3水平，专业任务

但代价也随之增加：
• 更大显存需求
• 更慢的推理速度
• 更高的训练成本

实践中需要平衡效果与成本。

如何选择合适的模型？

选择模型需要考虑多个因素：

1. 任务类型

任务类型 推荐模型 原因
简单对话 7B-13B模型 成本低，效果足够
代码生成 Claude、GPT-4、DeepSeek-Coder 编程能力是核心竞争力
长文档处理 Claude（200K上下文） 超长上下文窗口
中文任务 Qwen、DeepSeek 中文训练数据多，效果好
实时应用 小模型+量化 速度优先，牺牲精度
私有部署 Llama、Qwen等开源模型 数据安全，成本可控

2. 计算资源

  • 消费级显卡（如RTX 4090 24GB）：7B INT4量化、13B INT4量化

  • 无GPU（纯CPU）：小模型（7B以下）+ INT4量化
  • 云端API：使用闭源模型（GPT-4、Claude），无需本地GPU

第四章：训练

什么是训练？

训练（Training）是让模型从数据中学习的过程。通过反复调整模型参数，使其能准确完成任务。

类比理解

训练就像学生做练习题：
做题 → 对照答案 → 发现错误 → 调整方法 → 再做题 → ...
循环直到正确率足够高

训练的核心流程

准备数据
→
初始化模型
→
前向传播
→
计算损失
→
反向传播
→
更新参数

关键概念详解

1. 损失函数（Loss Function）

损失函数衡量模型预测与真实答案的差距。损失越小，模型越好。

  • 分类任务常用：交叉熵损失（Cross-Entropy）
  • 回归任务常用：均方误差（MSE）

2. 优化器（Optimizer）

优化器决定如何更新模型参数来降低损失。

  • SGD：随机梯度下降，最基础
  • Adam：自适应学习率，效果稳定，最常用

3. 学习率（Learning Rate）

学习率控制每次参数更新的幅度。

  • 太大：不稳定，可能跳过最优解
  • 太小：训练太慢
  • 需要调优找到合适值

批量大小（Batch Size）

一次训练使用多少样本：
• 小批量：更频繁更新，内存占用小
• 大批量：更新更稳定，但内存占用大

4. Epoch、Batch、Iteration

这三个概念描述训练的循环过程，新手容易混淆，务必理解清楚：

概念 定义 例子
Epoch（轮次） 整个数据集被完整学习一遍 1000个样本全部训练完=1个Epoch
Batch（批次） 一次训练使用的样本数量 每次用32个样本训练=Batch Size=32
Iteration（迭代） 训练一个Batch需要的步骤 1000样本÷32=31.25≈32次迭代

具体计算示例

假设：
• 数据集有1000张图片
• Batch Size = 32（每次训练32张）
• 要训练10个Epoch

计算：
• 每个Epoch需要：1000 ÷ 32 = 31.25 ≈ 32次迭代
• 10个Epoch总共需要：32 × 10 = 320次迭代
• 模型参数更新320次

训练时通常会看到进度条显示："Epoch 3/10, Batch 15/32"

为什么需要多个Epoch？

一次学习（1个Epoch）不足以让模型掌握知识，就像学生不能只看一遍课本就学会。需要多次重复学习，逐渐提高准确率。

  • Epoch太少：模型没学好，欠拟合
  • Epoch太多：模型"背熟了"，过拟合（记住训练数据，不能泛化）
  • 合适Epoch：观察训练曲线，找到损失不再明显下降的点

5. 梯度下降（Gradient Descent）详解

梯度下降是训练模型的核心算法，用于找到让损失最小的参数值。

山坡类比

想象你站在山坡上，目标是到达最低点（山谷）：
• 你看脚下，判断哪个方向最陡（梯度）
• 沿最陡方向往下走一步（参数更新）
• 重复：判断方向→走一步→判断方向→走一步
• 最终到达山谷（最优参数）

梯度就是山坡最陡的方向（函数变化最快的方向）
下降就是沿着这个方向往下走

梯度下降的三种方式

方式 每次使用的数据 特点 适用场景
批量梯度下降（BGD） 全部数据 方向准确，但计算慢 数据量小时使用
随机梯度下降（SGD） 单个样本 快但方向不稳定，易震荡 数据量大时使用
小批量梯度下降（Mini-batch SGD） 一小批样本（如32个） 平衡速度和稳定性，最常用 实际训练的主流方法
计算梯度
（损失函数对参数的导数）
→
确定方向
（梯度负方向）
→
更新参数
（参数 -= 学习率 × 梯度）
→
重复直到
损失足够小

梯度下降的关键公式

参数更新公式：
新参数 = 旧参数 - 学习率 × 梯度

解释：
• 梯度告诉我们"往哪里走能降低损失"
• 学习率告诉我们"走多远"
• 反复执行，最终找到最优参数

为什么学习率很重要？

学习率决定每一步走多远：

  • 学习率太大：一步跨太大，可能跳过最低点，来回震荡，无法收敛
  • 学习率太小：一步太小，下山太慢，训练时间很长
  • 合适的学习率：稳定下降，较快到达最优

学习率调优技巧

• 初始值通常：0.001（Adam默认）、0.01（SGD默认）
• 观察训练曲线：损失震荡→学习率太大；损失下降慢→学习率太小
• 可以动态调整：训练前期大学习率（快速下降），后期小学习率（精细调整）
• 常用策略：学习率衰减、余弦退火

6. 反向传播（Backpropagation）补充

反向传播是计算梯度的核心技术。它从输出层向输入层逐层计算每个参数对损失的影响。

为什么叫"反向"？

前向传播：数据从输入→隐藏层→输出（计算预测值）
反向传播：误差从输出→隐藏层→输入（计算梯度）

因为梯度计算是从输出层开始的，沿着网络结构"反向"传播误差信息，告诉每层"你哪里做错了，该怎么调整"。这就是神经网络的"自我纠错"过程。

训练的常见问题

问题 现象 解决方法
过拟合 训练数据表现好，新数据表现差 更多数据、正则化、Dropout
欠拟合 训练数据和新数据都表现差 增加模型复杂度、更多训练
梯度消失 深层网络无法有效学习 ReLU激活函数、Batch Normalization

第五章：推理

什么是推理？

推理（Inference）是使用已训练好的模型处理新数据，得到预测结果的过程。

简单理解

训练 = 制作工具
推理 = 使用工具

推理与训练的区别

阶段 目的 计算量 更新参数
训练 学习知识，调整参数 巨大（需要GPU集群） 会
推理 应用知识，输出结果 较小（单设备即可） 不会

推理的关键指标

  • 延迟（Latency）：处理单个请求的时间
  • 吞吐量（Throughput）：单位时间处理的请求数
  • 内存占用：运行模型需要的内存

推理部署方式

  1. 本地部署：模型在用户设备运行（如手机端AI）
  2. 云端部署：模型在服务器运行，用户通过API调用
  3. 边缘部署：模型在边缘设备运行（如智能摄像头）

推理优化技术详解

1. 量化（Quantization）——降低精度，减小模型

量化是将模型参数从高精度转换为低精度，减少内存占用和计算量。

为什么量化有效？

模型参数存储：
• FP32（32位浮点）：每个参数占4字节，精度高但占空间
• FP16（16位浮点）：每个参数占2字节，精度略降，节省一半
• INT8（8位整数）：每个参数占1字节，节省75%空间
• INT4（4位整数）：每个参数占0.5字节，节省87.5%空间

一个7B参数模型：
• FP32：约28GB显存
• INT8：约7GB显存
• INT4：约3.5GB显存

量化类型对比

量化方式 精度 模型大小 效果影响 适用场景
FP32（原始） 最高 100% 无损 科研、高精度要求
FP16/BF16 较高 50% 轻微下降 训练和推理常用
INT8量化 中等 25% 下降约1-2% 生产环境部署
INT4量化 较低 12.5% 下降约2-5% 边缘设备、消费级显卡

实际案例

Llama-2-70B模型：
• FP16版本：需要140GB显存（两张A100）
• INT4量化后：需要约35GB显存（一张A100或两张4090）
• 普通用户也能在家用显卡上运行大模型了！

2. 模型压缩技术

剪枝（Pruning）

剪枝是删除模型中不重要的参数（权重接近0的），减少模型复杂度。

  • 非结构化剪枝：删除单个参数，压缩效果好但推理加速有限
  • 结构化剪枝：删除整个神经元/层，推理速度明显提升

知识蒸馏（Knowledge Distillation）

知识蒸馏是让小模型学习大模型的知识，获得接近大模型的效果。

蒸馏过程

教师模型：大模型（如GPT-4），效果好但慢
学生模型：小模型（如蒸馏版），学习教师的输出

学生模型学习教师的"软标签"（概率分布），而非硬标签（正确答案）。
结果：小模型效果接近大模型，但速度快、内存小。

3. 推理加速技术

KV Cache（键值缓存）

KV Cache是LLM推理时缓存已计算的键值，避免重复计算。

为什么需要KV Cache？

LLM生成文本是逐词生成的：
• 生成"你"：计算所有词的注意力
• 生成"好"：再次计算所有词的注意力（包括"你"）
• 生成"啊"：又计算所有词的注意力...

问题：每次都要重复计算前面的词！
解决：KV Cache把前面的计算结果缓存起来，只计算新词，大幅加速推理。

Flash Attention

Flash Attention是优化注意力计算的算法，减少内存访问，提高速度。

  • 原理：将注意力计算分块处理，减少GPU显存读写次数
  • 效果：速度提升2-4倍，显存占用减少
  • 应用：几乎所有现代LLM训练和推理都使用

批处理推理

批处理是同时处理多个请求，提高吞吐量。

  • 单个请求处理效率低，GPU计算能力浪费
  • 批量处理（如10个请求一起）能充分利用GPU并行能力
  • 吞吐量大幅提升，但单个请求延迟可能略增

4. 推理框架对比

框架 特点 适用场景
vLLM PagedAttention，高效KV Cache管理 高吞吐量API服务
TensorRT-LLM NVIDIA优化，深度集成GPU NVIDIA显卡部署
ONNX Runtime 跨平台，CPU/GPU都支持 通用部署
llama.cpp C++实现，轻量级，支持CPU推理 本地部署、边缘设备

推理优化的权衡

• 量化：减小模型，但精度下降
• 批处理：提高吞吐，但延迟增加
• 剪枝：减小模型，但可能破坏结构

需要根据实际场景选择合适的优化方案，平衡效果、速度和资源。

第六章：RAG（检索增强生成）

什么是RAG？

RAG（Retrieval-Augmented Generation）是一种结合信息检索和生成模型的技术，让AI能利用外部知识库回答问题。

为什么需要RAG？

LLM的局限：
• 训练后知识固定，无法获取新信息
• 可能产生错误信息（幻觉）
• 无法访问私有数据

RAG解决：检索相关文档 → 增强上下文 → 生成准确回答

RAG的工作流程

用户提问
→
检索相关文档
→
构建增强上下文
→
LLM生成回答

RAG的核心组件

  1. 文档库：存储知识的外部数据源
  2. 向量数据库：存储文档的向量表示，支持相似检索
  3. Embedding模型：将文本转换为向量
  4. LLM：基于检索内容生成回答

RAG的优势

  • 回答更准确，减少幻觉
  • 能利用最新数据
  • 可引用数据来源
  • 适合企业私有知识库

常见向量数据库

Pinecone、Weaviate、Milvus、Chroma、Qdrant等
用于存储和检索文档向量

文档预处理与Chunking（分块）

Chunking是将长文档切分成小块的过程，是RAG系统的关键步骤。分块质量直接影响检索效果。

为什么需要分块？

• 向量维度限制：Embedding模型有最大输入长度（如512 tokens）
• 检索精度：小块更容易精准匹配查询
• 上下文窗口：LLM一次只能处理有限文本
• 语义完整性：避免在一个块中包含多个不相关主题

常见Chunking策略

策略 原理 优点 缺点
固定长度分块 按固定字符数/token数切分 简单易实现，块大小一致 可能切断语义完整性
语义分块 按段落、章节、句子边界切分 保持语义完整，理解更好 块大小不均匀，需额外处理
滑动窗口分块 相邻块有重叠部分（如重叠20%） 避免边界信息遗漏 增加存储和计算量
递归分块 先大块分割，再对大块细分 兼顾大块和小块检索 复杂度高，索引量大

分块参数建议

• 块大小：一般200-500 tokens（约500-1000字）
• 重叠量：推荐20-50 tokens重叠
• 保留元数据：每块记录来源文档、位置信息

示例：100页文档（约5万字）→ 分成约100块（每块500字）

Embedding模型选择

Embedding模型负责将文本转换为向量。不同模型有不同的特点和适用场景。

主流Embedding模型对比

模型 提供方 维度 特点 适用场景
text-embedding-ada-002 OpenAI 1536维 效果好，API调用方便 英文为主，商业应用
text-embedding-3-large OpenAI 3072维 最新版本，性能更强 高质量检索需求
BGE-large-zh BAAI（开源） 1024维 中文效果优秀，免费 中文场景，私有部署
M3E-large moka（开源） 1024维 中英文通用，效果稳定 中英混合文档

中文场景推荐

• 首选：BGE-large-zh（中文专用，效果最好）
• 次选：M3E-large（中英通用）
• 商业项目：OpenAI embedding（稳定可靠）
• 私有部署：开源模型本地运行，数据可控

相似度计算方法

检索时需要计算查询向量与文档向量的相似度。主要有三种方法：

方法 公式 特点 应用
余弦相似度 cos(A,B) = A·B / |A||B| 衡量方向相似，忽略大小 最常用，语义检索
欧氏距离 d(A,B) = √Σ(aᵢ-bᵢ)² 衡量绝对距离 图像检索、聚类
点积（Dot Product） A·B = Σaᵢbᵢ 计算简单，考虑大小 归一化向量的快速检索

余弦相似度通俗理解

两个向量的夹角越小，相似度越高：
• 夹角=0° → 相似度=1（完全相同方向）
• 夹角=90° → 相似度=0（完全不相关）
• 夹角=180° → 相似度=-1（完全相反）

例如：查询"人工智能"与文档"AI技术发展"的向量夹角小，相似度高；与文档"美食烹饪"的向量夹角大，相似度低。

检索策略详解

1. 纯向量检索

只使用向量相似度进行检索，适合语义匹配场景。

  • 优点：能找到语义相近的内容，不依赖关键词
  • 缺点：可能遗漏精确关键词匹配的内容
  • 适用：概念性问题、语义模糊的查询

2. 关键词检索（BM25）

使用传统关键词匹配算法，精确匹配查询词。

  • 优点：精确匹配关键词，速度快
  • 缺点：无法理解语义，同义词无法匹配
  • 适用：精确查询、专业术语搜索

3. 混合检索（Hybrid Search）

混合检索结合向量检索和关键词检索，是目前最佳实践。

用户查询
→
向量检索（语义匹配）
+
关键词检索（精确匹配）
→
合并结果+排序
→
返回最佳文档

混合检索的优势

• 同时匹配语义和关键词
• 覆盖更广，减少遗漏
• 对用户查询类型适应性更强
• 是企业级RAG系统的主流选择

4. 重排序（Re-ranking）

重排序是对初次检索结果进行精细化排序的二次处理。

  • 流程：向量检索返回Top 50 → 用Cross-Encoder精排 → 返回Top 5
  • Cross-Encoder：将查询和文档一起输入模型，计算精确相似度
  • 效果：提高最终结果的准确性，但计算成本更高
  • 适用：对检索精度要求高的场景

RAG优化技巧

HyDE（假设文档嵌入）

HyDE先让LLM生成一个假设的理想答案，再用假设答案去检索。

HyDE流程

用户查询："如何部署RAG系统？"
↓
LLM生成假设答案："部署RAG需要：1.准备向量数据库...2.配置Embedding模型..."
↓
用假设答案向量去检索（比原始查询更精准）
↓
返回真正相关的文档

Query改写

将用户的模糊查询改写为更清晰、更具体的检索查询。

Query改写示例

原查询："怎么用AI写代码"
改写后："AI代码生成工具的使用方法、配置步骤、最佳实践"

改写后的查询更详细，能匹配更多相关文档。

多路检索

用多种方式检索，合并结果。

  • 关键词检索：匹配精确术语
  • 向量检索：匹配语义相关
  • 知识图谱检索：匹配实体关系
  • 合并去重：综合多路结果，提高覆盖率

RAG vs 微调：如何选择？

RAG和微调都是让模型适应特定场景的方法，但适用情况不同。

对比维度 RAG 微调
知识来源 外部知识库实时检索 训练时固化在模型参数中
知识更新 更新知识库即可，无需重新训练 需要重新训练才能更新知识
成本 较低（向量存储+检索） 较高（GPU训练资源）
数据隐私 数据留在本地知识库 数据进入模型参数
响应速度 稍慢（需要检索步骤） 更快（直接推理）
适合场景 知识频繁更新、大量专业知识、需要引用来源 固定知识、特定风格/格式、高频重复任务

选择建议

优先使用RAG的情况：
• 企业知识库问答（政策、流程、产品信息）
• 需要引用来源、可追溯答案
• 知识需要频繁更新
• 数据隐私要求高

考虑微调的情况：
• 需要特定输出风格（如医学报告格式）
• 固定领域知识，不需要更新
• 对响应速度要求极高
• 有足够训练数据

最佳实践：RAG + 微调结合

• 用微调：让模型学会领域术语、输出格式
• 用RAG：提供最新的专业知识内容
• 结合使用，效果最佳

第七章：向量与标量

什么是标量？

标量（Scalar）是一个单独的数值，只有大小，没有方向。

  • 温度：25°C
  • 重量：5kg
  • 价格：100元

在AI中，单个参数、单个神经元输出都是标量。

什么是向量？

向量（Vector）是一组有序的数值，既有大小也有方向。

例子

二维向量：[3, 4] 表示坐标系中的点
三维向量：[1, 2, 3] 表示空间中的点
高维向量：[0.1, 0.2, ..., 0.768] 表示768维空间

向量在AI中的作用

Embedding（嵌入）是将文本、图像等转换为向量表示的技术。

为什么用向量？

向量能表示语义相似性：
• "猫"和"狗"的向量距离近（都是动物）
• "猫"和"汽车"的向量距离远
• 计算机通过向量理解语义关系

向量运算

运算 用途 AI应用
向量加法 合并信息 组合多个词的语义
向量点积 计算相似度 文档检索、推荐系统
向量余弦相似度 衡量方向相似 语义相似度计算

向量维度

  • 词向量：通常100-300维
  • BERT Embedding：768维
  • GPT-3 Embedding：12288维

维度越高，能表示的信息越丰富，但计算成本也越高。

第八章：神经网络核心概念

神经网络基础

神经网络是深度学习的基础架构，模拟人脑神经元连接的方式处理信息。

神经元（Neuron）

单个神经元的工作流程：

  1. 接收多个输入
  2. 加权求和（权重是可学习的参数）
  3. 加上偏置（bias）
  4. 通过激活函数
  5. 输出结果
输入 x₁, x₂, x₃
→
加权 w₁x₁ + w₂x₂ + w₃x₃
→
加偏置 +b
→
激活函数
→
输出 y

激活函数（Activation Function）

激活函数给神经网络引入非线性，使其能学习复杂模式。

函数 特点 用途
ReLU 简单高效：x>0输出x，否则输出0 最常用，深层网络
Sigmoid 输出0-1，适合概率 输出层分类
Tanh 输出-1到1 隐藏层
Softmax 多分类输出概率分布 多分类问题

反向传播（Backpropagation）

反向传播是训练神经网络的核心算法：

  • 前向传播：数据从输入层到输出层，得到预测
  • 计算损失：对比预测与真实值
  • 反向传播：从输出层向输入层计算梯度
  • 更新参数：根据梯度调整权重和偏置

第九章：常见模型架构

CNN（卷积神经网络）

CNN（Convolutional Neural Network）专门处理图像数据，能自动识别图像中的特征。它是图像识别、目标检测、人脸识别等视觉任务的核心技术。

为什么CNN适合处理图像？

图像是由像素组成的网格，相邻像素之间有关系（比如边缘是一条连续的线）。CNN能保留空间关系，不像传统方法把图像变成一串数字。它用"小窗口"扫描图像，就像人用眼睛逐区域观察一样。

CNN核心组件详解

1. 卷积层——特征提取的核心

卷积层是CNN最重要的部分。它使用滤波器（Filter/Kernel）在图像上滑动扫描，提取各种特征。

滤波器是什么？

滤波器是一个小矩阵（比如3×3），里面放着不同的数字权重。
• 边缘滤波器：能检测图像中的线条和边缘
• 纹理滤波器：能识别特定的纹理模式
• 颜色滤波器：能检测特定的颜色分布

一个卷积层可能有几十到几百个不同的滤波器，每个学习识别一种特征。

卷积操作的原理

想象你有一个小窗口（3×3像素），在图像上从左到右、从上到下滑动：

  1. 定位：窗口放在图像的某个位置
  2. 计算：窗口内的像素值与滤波器权重相乘，然后求和
  3. 输出：得到一个新的数值（特征值）
  4. 移动：窗口向右移动一格，重复上述过程
  5. 完成：扫描完整个图像，生成一张"特征图"

（224×224像素）
→

（多个滤波器扫描）

（每张对应一种特征）

特征提取的层次性

CNN通过多层卷积，逐层提取越来越复杂的特征：

层数 提取的特征 例子
浅层（第1-2层） 基础视觉元素 边缘、线条、简单颜色块
中层（第3-5层） 形状和部件 圆形、方形、眼睛形状、轮子形状
深层（第6-N层） 完整物体 人脸、汽车、猫、狗的整体形态

实例：CNN如何识别一张猫的照片

第1层卷积：检测到边缘线条 → 发现猫轮廓的边界
第2层卷积：检测到形状 → 发现圆形的眼睛、三角形的耳朵
第3层卷积：检测到部件 → 眼睛+耳朵+胡须的组合
第4层卷积：检测到整体 → "这像是猫的脸"
全连接层：综合判断 → 输出"这是猫（概率98%）"

2. 池化层——降维与关键特征保留

池化层的作用是缩小特征图的尺寸，同时保留最重要的特征信息。

为什么需要池化？

• 减少计算量：特征图太大，计算成本高
• 防止过拟合：去除过多细节，保留核心特征
• 增强鲁棒性：即使图像稍微移动、缩放，关键特征仍能被识别

例如：一张224×224的特征图，池化后变成112×112，信息量减少但关键特征保留

常见池化方式

池化类型 原理 特点
最大池化（Max Pooling） 在区域内取最大值 保留最强特征，最常用
平均池化（Average Pooling） 在区域内取平均值 保留整体趋势，较平滑

3. 全连接层——最终分类决策

全连接层将前面提取的所有特征综合起来，做出最终判断。

  • 接收输入：来自最后一层卷积/池化的特征
  • 综合判断：将所有特征"投票"决定类别
  • 输出结果：每个类别的概率值（如：猫90%、狗5%、鸟5%）

CNN的局限

• 主要处理图像，不适合序列数据（文本、语音）
• 对图像旋转、翻转敏感（需要数据增强）
• 需要大量标注数据训练
• 计算资源需求较大

Transformer

Transformer是目前最主流的AI架构，GPT、BERT、Claude等几乎所有大语言模型都基于它。2017年由Google提出，彻底改变了自然语言处理领域。

Transformer的革命性突破

之前的模型（RNN）需要逐词处理，慢且难并行。
Transformer可以同时处理整个句子，速度快、效果好。
这让大模型训练成为可能，开启了AI新时代。

Transformer的核心：注意力机制

注意力机制（Attention）让模型能关注输入中最相关的部分，就像人读书时会重点关注某些词句。

通俗理解

阅读句子"我喜欢编程，特别是Python"
处理"Python"这个词时，注意力会关注"编程"
因为它们语义相关，"Python"是"编程"的一种

这就是注意力机制的核心思想：根据当前词，找到相关词，重点参考。

Self-Attention（自注意力）详解

自注意力让序列中的每个词都能关注其他所有词，获取上下文信息。

Query、Key、Value的比喻

图书馆类比

想象你在图书馆找书：
• Query（查询）：你想要找什么内容（当前词的需求）
• Key（标签）：每本书的标签/目录（其他词的标识）
• Value（内容）：书里的实际内容（其他词的信息）

你的Query与每本书的Key对比，找到最匹配的书，取出Value（书的内容）。相似度越高，这本书对你的价值越大。

自注意力计算过程

  1. 创建向量：每个词生成Query、Key、Value三个向量
  2. 计算相似度：Query与所有Key的点积，得到注意力分数
  3. 归一化：分数除以√d（防止数值太大），再通过Softmax变成概率
  4. 加权求和：用概率权重对所有Value加权，得到新的词表示
输入："我 爱 学习"
→
每个词生成Q/K/V
→
"学习"的Q与其他词的K匹配
→
得到注意力权重
→
加权组合得到新表示

多头注意力（Multi-Head Attention）

多头注意力是让模型从多个不同角度理解句子。

为什么需要"多头"？

一个注意力头只能关注一种关系：
• Head 1 可能关注语法关系（主语-谓语）
• Head 2 可能关注语义关系（名词-属性）
• Head 3 可能关注位置关系（上下文）

多个头可以同时关注不同类型的关系，理解更全面。
GPT-3使用了96个注意力头！

多头注意力工作原理

  • 把Query、Key、Value分成多份（比如8份）
  • 每份独立做一次自注意力计算
  • 得到8个不同的"视角"结果
  • 把8个结果合并，得到最终表示

位置编码（Positional Encoding）

位置编码告诉模型每个词在句子中的位置，弥补Transformer无法识别顺序的缺陷。

为什么需要位置编码？

Transformer同时处理所有词，不知道"我吃苹果"和"苹果吃我"的区别（词相同，顺序不同）。

位置编码给每个词添加一个"位置标签"，让模型能区分：
"我"（位置1）+ "吃"（位置2）+ "苹果"（位置3）

这样模型就知道词的顺序，理解正确含义。

Transformer架构详解

Transformer由两个主要部分组成：

Encoder（编码器）——理解输入

Encoder负责理解输入文本，提取其中的信息。

  • 输入：原始文本（如"把这段话翻译成英语"）
  • 处理：多层自注意力+前馈网络，理解每个词及其上下文
  • 输出：每个词的深层语义表示
  • 代表模型：BERT（双向理解，适合分类、问答）

Decoder（解码器）——生成输出

Decoder负责生成输出文本，逐词预测下一个词。

  • 输入：Encoder的输出 + 已生成的词
  • 处理：自注意力（看已生成部分）+ 交叉注意力（看Encoder输出）
  • 输出：下一个词的概率分布，选择最可能的词
  • 代表模型：GPT（单向生成，适合写作、对话）

不同架构的应用场景

架构 模型 特点 适合任务
仅Encoder BERT、RoBERTa 双向理解整个文本 文本分类、情感分析、问答系统
仅Decoder GPT、LLaMA、Qwen 单向生成，预测下一个词 写作、对话、代码生成
Encoder-Decoder T5、BART 理解+生成结合 翻译、摘要、改写

关键创新总结

Transformer的三大创新：
• 自注意力：词与词直接关联，理解全面
• 多头机制：多个角度分析，更丰富
• 并行计算：同时处理，速度快

这三点让Transformer成为现代AI的基石。

• 编程问题：temperature=0.2
• 创意写作：temperature=0.8
• 一般对话：temperature=0.5

第十二章：工具（Tools）与技能（Skills）

什么是工具？

工具（Tools）是Agent可以调用的外部功能，让AI能执行实际操作，而不仅仅是生成文本。

为什么需要工具？

LLM本身只能生成文本，无法：
• 搜索最新信息
• 执行代码
• 访问数据库
• 操作文件

工具让AI突破这些限制

常见工具类型

工具类型 功能 应用场景
搜索工具 搜索网络信息 获取实时数据、查找资料
代码执行 运行Python等代码 数据分析、计算、自动化
文件操作 读写文件 文档处理、数据导入导出
API调用 调用外部服务 获取天气、发送邮件、数据库查询
知识检索 从知识库检索 RAG、企业知识查询

Function Calling（函数调用）

函数调用是LLM调用外部工具的技术机制。

用户请求
→
LLM判断需要工具
→
生成函数调用参数
→
执行函数
→
返回结果给LLM
→
生成最终回答

函数调用示例

用户："今天北京天气怎么样？"
LLM：调用get_weather(city="北京")
函数返回：{"temperature": 25, "weather": "晴"}
LLM："北京今天天气晴朗，温度25°C"

什么是技能（Skills）？

技能（Skills）是AI Agent的特定能力模块，类似于人类的"专业技能"。

类比理解

人：掌握多个技能（编程、写作、设计）
Agent：配置多个技能模块（代码生成、数据分析、文档撰写）

技能的特点

  • 模块化：可以独立配置和启用/禁用
  • 专业化：针对特定领域优化
  • 可扩展：可以添加新技能
  • 可组合：多个技能协作完成任务

常见技能示例

技能领域 具体技能 应用
编程技能 代码生成、调试、重构 软件开发、自动化脚本
分析技能 数据分析、图表生成、报告撰写 商业分析、科学研究
写作技能 文章创作、编辑润色、翻译 内容创作、文档处理
搜索技能 网络搜索、信息提取、知识检索 调研、信息收集
设计技能 图像生成、UI设计建议 创意设计、产品开发

技能 vs 工具

• 工具：底层能力，如"搜索"、"执行代码"
• 技能：上层组合，整合工具+知识+策略

例如："数据分析技能"包含：
- 代码执行工具 + 统计知识 + 分析流程策略

第十三章：知识库与向量数据库

什么是知识库？

知识库（Knowledge Base）是存储结构化或非结构化知识的系统，让AI能访问专业知识、企业数据、实时信息。

为什么需要知识库？

LLM的问题：
• 知识有限（训练数据截止）
• 无法访问企业私有数据
• 可能产生错误信息（幻觉）

知识库解决：
• 提供最新、准确的专业知识
• 让AI"知道"企业内部信息

知识库的类型

类型 特点 应用
文档库 PDF、Word、网页等文档集合 企业文档查询、政策解读
结构化数据库 SQL数据库，有明确结构 客户信息、订单数据、产品目录
向量数据库 存储向量表示，支持语义搜索 RAG、相似内容检索
知识图谱 实体和关系的网络结构 复杂关系查询、推理

知识库建设流程

收集数据
→
清洗处理
→
分块切分
→
向量编码
→
存储入库
→
检索使用

向量数据库详解

向量数据库是专门存储和检索向量Embedding的数据库，支持语义相似性搜索。

传统搜索 vs 向量搜索

搜索方式 原理 特点
关键词搜索 匹配文字相同 "苹果"只能找到包含"苹果"的文档
向量搜索 匹配语义相似 "苹果"能找到"水果"、"iOS设备"等相关内容

向量搜索的优势

• 理解语义，不是简单匹配文字
• 能找到意思相近的内容
• 支持跨语言搜索
• 处理模糊、抽象的查询

常见向量数据库

  • Pinecone：云端托管，易于使用，适合生产环境
  • Weaviate：开源，支持多种数据类型，功能丰富
  • Milvus：开源，高性能，适合大规模场景
  • Chroma：轻量级开源，适合开发测试
  • Qdrant：开源，高效，支持过滤

向量检索流程

  1. 查询向量化：将用户查询转换为向量
  2. 相似度计算：与库中向量计算距离
  3. 排序筛选：返回最相似的N个结果
  4. 返回原文：返回对应的原始文本

第十五章：高级提示技术

Chain of Thought（思维链）

思维链是一种让LLM逐步展示推理过程的提示技术，能显著提高复杂问题的准确率。

基本方法

在Prompt中加入："让我们一步步思考"
或者："请展示你的推理过程"

思维链的效果

问题
→
步骤1：理解问题
→
步骤2：分解任务
→
步骤3：逐步推理
→
步骤4：验证答案
→
最终结论

思维链的应用

  • 数学推理：逐步计算，减少错误
  • 逻辑分析：展示推理过程
  • 复杂决策：权衡多个因素
  • 问题诊断：系统性排查

Zero-shot vs Few-shot

零样本（Zero-shot）：不给示例，让LLM直接完成任务。

少样本（Few-shot）：提供几个示例，让LLM学习模式。

示例对比

Zero-shot：
"将以下句子翻译成法语：你好"

Few-shot：
"将以下句子翻译成法语：
你好 → Bonjour
谢谢 → Merci
再见 → Au revoir
我爱你 → ?"

何时使用Few-shot

  • 任务格式特殊，需要示范
  • 输出风格需要统一
  • LLM对任务不够熟悉
  • 需要特定格式输出

ReAct（推理+行动）

ReAct是让Agent交替进行推理和行动的技术，边思考边执行。

ReAct流程

  1. Thought：思考下一步做什么
  2. Action：执行具体操作
  3. Observation：观察结果
  4. 循环：重复直到完成

ReAct示例

任务："查找北京今天的天气"
Thought: 我需要搜索北京的天气信息
Action: search("北京天气")
Observation: 北京今天晴，25°C
Thought: 我已获得答案
Action: finish("北京今天天气晴朗，温度25°C")

Tree of Thoughts（思维树）

思维树（ToT）是让LLM在解决问题时探索多个推理分支，选择最优路径的技术。它将思维链扩展为树状结构，能处理需要回溯和探索的复杂问题。

为什么需要思维树？

思维链（CoT）是线性推理：
问题 → 步骤1 → 步骤2 → 答案

思维树（ToT）是分支探索：
问题 → 步骤1a → 步骤2a → 步骤3a → 答案A
　　　 → 步骤1b → 步骤2b → 步骤3b → 答案B
　　　 → 步骤1c → 步骤2c → 步骤3c → 答案C

然后评估哪个答案最优。

思维树的工作流程

问题定义
→
生成多个初始思路
→
评估每个思路
→
选择最优思路
→
继续分支扩展
→
最终答案

思维树的实现方式

方法 描述 适用场景
广度优先搜索（BFS） 每层探索所有分支，选择最优继续 需要全面评估的问题
深度优先搜索（DFS） 深入探索一条路径，不通则回溯 需要深度推理的问题
启发式搜索 使用评估函数指导搜索方向 有明确评估标准的问题

思维树示例：24点游戏

问题：用1、3、5、7四个数字，通过加减乘除得到24

分支1：尝试 (1+3) × (5+7) = 4 × 12 = 48 ❌ 太大
分支2：尝试 (7-5) × (1+3) = 2 × 4 = 8 ❌ 太小
分支3：尝试 (1+5) × (7-3) = 6 × 4 = 24 ✅ 正确！

通过探索多个分支，找到正确答案。

思维树的代价

• 计算成本更高：需要生成多个分支，调用多次LLM
• 评估困难：如何判断哪个分支更好？
• 适用场景有限：简单问题不需要分支探索

建议：只在复杂推理、需要多方案对比时使用

Self-Consistency（自一致性）

自一致性是让LLM多次生成答案，然后通过投票选择最一致的结果，能显著提高准确率。

核心思想

• 单次回答可能出错，但多次回答中"正确的答案"更容易出现
• 如果同一类答案出现多次，说明这个答案更可靠
• 类似于"群众的智慧"

Self-Consistency流程

同一问题多次提问
→
收集N个答案
→
统计答案分布
→
选择出现最多的结果
→
最终答案

Self-Consistency示例

问题：一个商店原价100元，打8折后再打9折，最终价格是多少？

多次回答（带思维链）：

回答1：先打8折：100×0.8=80元，再打9折：80×0.9=72元 → 72元
回答2：100×0.8=80，80×0.9=72 → 72元
回答3：100×0.72=72元 → 72元
回答4：100×0.8=80，80×0.9=72 → 72元

投票结果：72元出现4次 → 最终答案：72元

Self-Consistency实现要点

  • 采样次数：通常5-10次，太多成本高，太少不够稳定
  • 温度设置：用较高温度（0.7-1.0）增加答案多样性
  • 投票策略：多数投票（Majority Vote）或加权投票
  • 适用任务：数学计算、逻辑推理、分类任务

成本vs收益

• 准确率提升：约5-15%（取决于任务难度）
• 计算成本：增加N倍（N为采样次数）
• 建议场景：对准确率要求高的任务（如医疗、金融决策）
• 不建议场景：简单任务、成本敏感场景

CRISPE框架

C - Capacity（角色）：AI应该扮演什么角色
如："你是一位资深Python工程师"

R - Request（请求）：你需要AI做什么
如："请帮我调试这段代码"

I - Insight（背景信息）：提供必要的上下文
如："这段代码应该计算列表平均值，但结果不正确"

S - Style（风格）：期望的输出风格
如："请详细解释错误原因，并给出修正后的代码"

P - Personality（个性）：AI的表达方式
如："用通俗易懂的语言，适合初学者"

E - Experiment（实验）：可以尝试多个版本
如："提供3种可能的解决方案"

第十七章：数据——AI的燃料

什么是数据集？

数据集（Dataset）是用于训练、验证和测试AI模型的数据集合，是AI学习的"教材"。

简单理解

就像学生需要课本学习一样，AI需要数据集来学习。
数据集质量越高，AI学得越好。

数据集的三种划分

类型 比例 作用 类比
训练集 70-80% 模型学习知识 课本，用于学习
验证集 10-15% 调优模型参数 模拟考试，检验学习效果
测试集 10-15% 最终评估性能 期末考试，最终评分

重要原则

测试集绝对不能用于训练！否则就像"提前看过期末考试题"，评估结果不真实。

数据标注

数据标注是为原始数据添加标签（正确答案）的过程，是监督学习的前提。

标注的类型

  • 分类标注：给图片打标签（"猫"、"狗"）
  • 边界框标注：标注物体位置（目标检测）
  • 文本标注：标注情感、意图、实体
  • 语音标注：转录文字、标注说话人

标注的挑战

• 标注成本高：需要大量人力
• 标注质量关键：错误标注会误导模型
• 专业标注难：医学图像等需要专家知识

数据清洗

数据清洗是处理原始数据中的错误、缺失、重复等问题，提高数据质量。

常见数据问题

  • 缺失值：部分数据为空
  • 重复数据：同一条数据多次出现
  • 错误数据：标注错误、格式错误
  • 噪声数据：无关信息、干扰信息
  • 不平衡数据：某些类别数据过多/过少

清洗方法

问题 处理方法
缺失值 填充默认值、删除、预测填充
重复数据 去重处理
错误数据 修正或删除
噪声数据 过滤、平滑处理
不平衡数据 数据增强、重采样

数据增强

数据增强是通过变换现有数据生成新数据，增加数据量的技术。

图像数据增强示例

原始图片 → 翻转、旋转、缩放、裁剪、调整亮度 → 生成多张新图片
一张图片可以变成几十张训练数据

为什么需要数据增强

  • 数据量不足时，增加训练样本
  • 提高模型鲁棒性，适应各种变化
  • 解决数据不平衡问题
  • 降低标注成本

第十八章：模型评估

为什么需要评估？

训练完成后，需要评估模型性能，判断模型是否达到要求，是否可以投入使用。

核心问题

• 模型准确吗？
• 模型稳定吗？
• 模型能处理各种情况吗？

核心评估指标

准确率（Accuracy）

准确率是模型预测正确的比例。

公式

准确率 = 预测正确的数量 / 总预测数量
例如：100个样本，预测正确90个，准确率=90%

精确率与召回率

这两个指标用于评估分类模型的不同角度：

指标 含义 应用场景
精确率（Precision） 预测为正的样本中，真正为正的比例 "我说对的，有多少真的对"
召回率（Recall） 真正的正样本中，被预测为正的比例 "真的对的，我找出来多少"

精确率vs召回率权衡

• 高精确率：宁可漏过，不可错报（医疗诊断）
• 高召回率：宁可错报，不可漏过（垃圾邮件检测）
• 通常需要根据场景权衡

F1分数

F1分数是精确率和召回率的调和平均，综合评估模型性能。

当精确率和召回率都很重要时，使用F1分数作为综合指标。

损失值（Loss）

损失值衡量模型预测与真实答案的差距，训练过程中持续下降说明模型在学习。

  • 训练初期：损失值高，模型在学习
  • 训练中期：损失值下降，模型进步
  • 训练后期：损失值稳定，模型收敛

查看训练曲线

通过绘制"损失值曲线"可以观察：
• 学习是否正常进行
• 是否过拟合（训练损失降，验证损失升）
• 训练是否需要停止

混淆矩阵

混淆矩阵是展示分类结果的表格，清晰显示预测正确和错误的情况。

预测为正 预测为负
实际为正 真正例（TP） 假负例（FN）
实际为负 假正例（FP） 真负例（TN）

从混淆矩阵可以计算：
• 准确率 = (TP + TN) / 总数
• 精确率 = TP / (TP + FP)
• 召回率 = TP / (TP + FN)

第十九章：GPU与算力

什么是算力？

算力（Computing Power）是计算机处理数据的能力，用FLOPS（每秒浮点运算次数）衡量。

算力单位

• TFLOPS：每秒万亿次运算
• PFLOPS：每秒千万亿次运算
• EFLOPS：每秒百亿亿次运算

GPT-3训练约需要3.64 × 10²³ FLOPS

CPU vs GPU

GPU（图形处理器）是AI训练的核心硬件，比CPU更适合大规模计算。

特性 CPU GPU
设计目的 通用计算，逻辑处理 图形渲染，并行计算
核心数量 几十个 数千个
擅长任务 串行任务、复杂逻辑 并行任务、矩阵运算
AI训练 慢，不适合大规模训练 快，是AI训练主流

类比理解

CPU：像几个教授，能处理复杂问题
GPU：像几千个小学生，能同时做简单计算

神经网络训练需要大量简单计算，GPU更合适

为什么AI训练需要GPU

  • 并行计算：神经网络有大量矩阵运算，GPU能同时处理
  • 效率高：GPU训练速度比CPU快几十到几百倍
  • 成本合理：同样的计算量，GPU更经济

主流GPU产品

GPU型号 特点 应用
NVIDIA RTX 4090 消费级最强，24GB显存 个人开发、小模型训练
NVIDIA A100 数据中心级，80GB显存 大模型训练、企业应用
NVIDIA H100 最新旗舰，专为AI优化 GPT-4级别模型训练

显存的重要性

显存是GPU的内存，决定了能处理多大的模型。

为什么显存重要

模型参数需要存储在显存中：
• 7B参数模型：约需14GB显存
• 13B参数模型：约需26GB显存
• 70B参数模型：约需140GB显存

显存不够，就无法训练或运行大模型

显存优化技术

  • 量化：减少参数精度，降低显存需求
  • 模型分片：将模型分散到多个GPU
  • 梯度累积：小批量多次累积，模拟大批量

第十六章：AI局限与挑战

幻觉问题（Hallucination）

幻觉是LLM生成虚假、不准确信息的现象，是AI应用的主要风险。

幻觉的类型

类型 表现 风险
事实幻觉 编造不存在的事实 "2025年火星已有人居住"
来源幻觉 引用不存在的来源 "根据《AI发展报告2024》..."
数字幻觉 编造统计数据 "研究表明85%的用户..."
逻辑幻觉 推理过程出错 错误的因果结论

减少幻觉的方法

  • 使用RAG：检索真实信息增强回答
  • 要求引用来源：让AI说明信息出处
  • 人工验证：重要信息需要核实
  • 限定范围：指定只使用已知信息
  • 多模型交叉验证：对比不同模型输出

安全建议

• 不要完全信任AI的医疗、法律建议
• 重要决策前核实关键信息
• 教育用户了解AI局限性
• 建立信息审核机制

对齐问题（Alignment）

对齐是让AI行为符合人类价值观和期望的技术挑战。

对齐的维度

  • 指令对齐：AI正确理解用户意图
  • 价值对齐：AI行为符合道德伦理
  • 安全对齐：AI不产生有害内容
  • 真实对齐：AI提供准确信息

RLHF（基于人类反馈的强化学习）

RLHF是训练AI对齐的主要方法：

  1. 预训练：学习语言能力
  2. 监督微调：学习遵循指令
  3. 奖励模型：人类评价AI输出质量

多模态（Multimodal）

多模态AI能同时处理多种类型的数据：文本、图像、音频、视频等。

多模态的优势

  • 理解更丰富：结合视觉和语言信息
  • 应用更广：图像分析、视频理解、语音识别
  • 交互更自然：支持多种输入方式

常见多模态模型

模型 能力 应用
GPT-4 Vision 理解图像内容 图像问答、视觉分析
Claude 3 文本+图像处理 文档分析、图表解读
Gemini 文本+图像+视频+音频 全面多模态应用
